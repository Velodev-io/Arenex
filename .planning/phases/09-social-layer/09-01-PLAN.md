# Plan: 09-01 - Anonymous Social Layer (Likes & Comments)

**Wave:** 1
**Depends on:** Nothing
**Files modified:** app/models/__init__.py, app/main.py, frontend/src/pages/Spectator.jsx
**Autonomous:** true

## Description
This plan implements anonymous match interaction (Likes and Comments) with persistent storage and UI display.

## Tasks

<task read_first="app/models/__init__.py">
<action>
Add `Comment` and `MatchLike` models:
- `Comment`: id, match_id, display_name, content, created_at.
- `MatchLike`: id, match_id, created_at.
Update `Match` model to include relationships.
</action>
<acceptance_criteria>
- Models are defined in `app/models/__init__.py`.
</acceptance_criteria>
</task>

<task read_first="app/main.py">
<action>
Create `app/api/social.py` with endpoints:
- `POST /social/match/{id}/like`: Increments likes.
- `POST /social/match/{id}/comment`: Payload { "display_name": str, "content": str }.
- `GET /social/match/{id}/stats`: Returns { "likes": int, "comments": list }.
Register router in `app/main.py`.
</action>
<acceptance_criteria>
- Endpoints are functional and return 200 via `curl`.
</acceptance_criteria>
</task>

<task read_first="frontend/src/pages/Spectator.jsx">
<action>
Enhance Spectator/Replay UI:
- Add Heart icon with total count.
- Add "Social Feed" sidebar section.
- Add input form for anonymous comments (Display Name + Text).
</action>
<acceptance_criteria>
- Users can click like and see the count update.
- Comments are fetched and displayed in a scrolling list.
</acceptance_criteria>
</task>
