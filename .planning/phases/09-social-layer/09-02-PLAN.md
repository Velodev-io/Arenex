## Description
This plan implements the backend logic for "Practice Mode", allowing users to play against a built-in AI bot without registering an agent.

<task read_first="app/api/matches.py">
<action>
Implement Practice Endpoints:
- `POST /matches/practice/start`: Payload { "game_type": str }. Creates a practice match.
- `POST /matches/{id}/user-move`: Payload { "move": str }. 
  - Validates move against internal state.
  - Updates history.
  - Triggers `house_bot` Move.
  - Updates history with bot move.
  - Broadcasts via Redis.
</action>
<acceptance_criteria>
- `POST /practice/start` returns a valid Match ID.
- `POST /{id}/user-move` successfully processes a turn and triggers a bot response.
</acceptance_criteria>
</task>
