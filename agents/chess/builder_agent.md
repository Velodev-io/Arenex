# Builder Agent Spec

## Overview
This agent is a standalone Python script designed to continuously author and improve the `chess_agent.py` FastAPI server code. It is Phase 2 of the Chess Agent pipeline.

It utilizes the **Groq API** (using models like `llama-3.3-70b-versatile`) to generate Python code. It can operate in two modes:
1.  **Initial Run:** It takes a comprehensive System Prompt detailing the chess agent constraints and priority rules, and generates `chess_agent.py` entirely from scratch.
2.  **Improvement Run:** It consumes a JSON "failure report" generated by the Tester Agent, dynamically incorporates those failures into the user prompt, and asks the LLM to rewrite the code to fix the specific weaknesses without regressing on passing behaviors.

## Dependencies
- `groq` (for making API calls to Groq)
- `json`, `os`, `re`, `argparse` (standard Python libraries)

## Core Logic and Flow

1.  **Environment Check:** The script immediately checks for the existence of the `GROQ_API_KEY` environment variable. It raises an error and halts if absent.
2.  **Report Parsing (Optional):** It checks CLI arguments for an optional `--report <path_to_json>` flag. If provided, it reads the Tester Agent's failure report.
3.  **Prompt Construction:**
    *   **System Prompt (Constant):** An extensive string explicitly dictating the rules the generated `chess_agent.py` MUST follow. It enforces:
        *   The exact `POST /move` and `GET /health` REST contract.
        *   *Mandatory* use of `python-chess` for move logic, validation, parsing, and piece values (P=1, N=3, B=3, R=5, Q=9, K=1000).
        *   The 10-step Priority Strategy ladder (Win, Checkmate, Check, High-Value Capture, Protect, Center Control, Develop, Castle, Avoid re-moving, Random).
        *   Output constraint: ONLY raw Python code wrapped in ` ```python ... ``` ` blocks, with no extraneous conversational text.
    *   **User Prompt (Dynamic):**
        *   *If no report:* Requests the complete file from scratch implementing all 10 priorities.
        *   *If report provided:* Injects the `pass_rate`, `identified_weaknesses`, raw `failed_puzzles` JSON, and `game_results` from the report into the prompt. It instructs the LLM to rewrite the file to fix those precise weaknesses while maintaining passing features.
4.  **API Invocation:** Sends the System and User prompts to the Groq API using `groq.Groq`. It targets the `llama-3.3-70b-versatile` model.
5.  **Output Extraction:** Applies a Regex pattern ` ```python\s*(.*?)``` ` to isolate the raw source code from the LLM's response.
6.  **File Generation:** Overwrites the local `chess_agent.py` file (or the configured output file) with the extracted Python string.

## Output Structure

The output is usually a generated `chess_ai_server.py` file (or similar) residing in the same directory.

## How to Run

Ensure the Groq API key is exported:
```bash
export GROQ_API_KEY="gsk_..."
```

Run from scratch (Initial generation):
```bash
python3 chess_builder.py
```

Run with feedback from the tester (Improvement iteration):
```bash
python3 chess_builder.py --report tester_report.json
```
