# Tester Agent Spec

## Overview
This agent is a standalone Python test suite that validates the robustness and logic of a running `chess_agent.py` server. It acts as Phase 3 of the Chess Agent pipeline.

It communicates with the chess agent exclusively over HTTP, sending board states and evaluating the resulting moves. It compiles its findings into a highly structured JSON report.

## Dependencies
- `requests` (for HTTP communication)
- `python-chess` (for board state management, parsing, and move validation)
- `json`, `argparse`, `time`, `random` (standard Python libraries)

## Test Suites

The Tester executes three distinct suites, evaluating both logical correctness and systemic stability:

### Suite 1: Tactical Puzzles (Logic Verification)
*   **Method:** Evaluates the agent against an array of 10 hardcoded tactical scenarios defined by FEN strings.
*   **Scenarios Tested:** Forced checkmates (mate in 1), capturing hanging/free pieces, saving own hanging pieces, choosing high-value captures over low-value ones, castling effectively, center control, avoiding illegal moves, avoiding self-check, checking to fork.
*   **Evaluation:** Binary pass/fail. The agent's response (`move` from the API) is checked against a predefined list of acceptable UCI moves for that exact scenario.

### Suite 2: Games vs Random Opponent (Competency check)
*   **Method:** Plays 5 full chess games (max 200 moves) against the tester.
*   **Opponent Logic:** The tester chooses completely random legal moves.
*   **Evaluation:** A rule-based agent following basic priorities (like not hanging pieces and taking free pieces) should statistically beat random moves 100% of the time. The expected outcome is 5 wins, 0 losses. A loss indicates a deep fatal logic flaw. Crashes, timeouts, or illegal move attempts are recorded as failures.

### Suite 3: Mirror Games (Stability Check)
*   **Method:** Plays 3 full games (max 200 moves) where the agent is pitted against itself (the tester queries the agent for both white and black turns).
*   **Evaluation:** This suite primarily tests for systemic issues rather than wins/losses. It monitors for:
    *   **Infinite loops/Stalemates:** Tracks FEN strings to detect threefold repetition.
    *   **Timeouts:** Ensures the agent doesn't dead-lock when playing against its own strategy.
    *   **Crashes:** Validates the agent returns legal moves even in deep, complex mid-games generated by its own logic.

## Logic and Flow

1.  **Server Polling:** Iteratively polls `GET /health` until the agent server is ready or a timeout occurs.
2.  **Execute Suites:** Runs Suite 1, Suite 2, and Suite 3 sequentially, collecting telemetry, win/loss records, and failed puzzle payloads.
3.  **Inference Engine:** Parses the aggregated failures and translates them into an `identified_weaknesses` array of human/LLM-readable strings (e.g., "Agent failed 3 'hanging_queen' puzzles", "Agent timed out in 2 mirror games").
4.  **Report Generation:** Calculates a consolidated `pass_rate` (percentage of passed puzzles + successful games against total test points) and saves a structured JSON report to disk.

## Output JSON Format (`tester_report.json`)

```json
{
  "pass_rate": 0.75,
  "threshold": 0.8,
  "passed": false,
  "failed_puzzles": [
    {
      "scenario": "mate_in_one",
      "description": "Mate in one — back rank checkmate",
      "board_fen": "6k1/5ppp/8/8/8/8/8/R5K1 w - - 0 1",
      "expected_move": "a1a8",
      "agent_move": "a1b1",
      "reasoning_given": "Developing rook"
    }
  ],
  "game_results": {
    "vs_random": { "wins": 4, "losses": 1, "draws": 0, "crashes": 0, "timeouts": 0 },
    "vs_mirror": { "crashes": 0, "timeouts": 1, "loops": 0 }
  },
  "identified_weaknesses": [
    "Agent failed 1 'mate_in_one' puzzle(s)",
    "Agent lost 1/5 games vs random opponent (should win 100%)",
    "Agent timed out in 1/3 mirror games"
  ]
}
```

## How to Run

Targeting the default local agent server:
```bash
python tester_agent.py
```

Override defaults via CLI flags:
```bash
python tester_agent.py --url http://127.0.0.1:8001 --output results.json
```
